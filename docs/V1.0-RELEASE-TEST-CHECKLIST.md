# Movie Schedule Alarm V1.0 Release 測試清單

測試人員應記錄日期、Windows 版本、測試檔名、Portable／Setup 類型與實際結果。任何未執行項目不得標示通過。

## 一、自動檢查

- [ ] `npm.cmd run test:regression` 通過。
- [ ] 全部 JavaScript `node --check` 通過。
- [ ] `git diff --check` 通過。
- [ ] 全專案只有 `js/countdown.js` 建立一個 `setInterval`。
- [ ] 全專案只有 `js/alarm.js` 建立一個 Audio Channel。
- [ ] 完整 application state 只存在於 `js/app.js`。

## 二、Excel 與 PDF 匯入

- [ ] 匯入實際 Excel，日期、開始、結束、影廳、片名、語言及規格正確。
- [ ] 斷網後仍可匯入 Excel。
- [ ] 匯入實際單日 PDF，場次數與原始報表相符。
- [ ] 匯入實際整週 PDF，日期下拉選單完整。
- [ ] 損壞月份標題可由檔名週期安全修復；無法唯一判定時拒絕猜測。
- [ ] 掃描圖片型 PDF 顯示非阻塞錯誤且保留上一份場次。
- [ ] 匯入失敗不清除上一份有效資料或警報排程。

## 三、時間、跨午夜與營運日

- [ ] 00:00–05:59 場次歸入前一營運日，但實際日期標題保持正確。
- [ ] 06:00 後場次歸入當日營運日。
- [ ] 結束時間跨午夜時 finishDateTime 正確。
- [ ] 多日場次依完整日期時間排序。
- [ ] 週期最後一場結束後只提醒一次匯入新週期。

## 四、Next Movie、倒數與狀態

- [ ] Next Movie 不受搜尋、日期、影廳、語言、規格及狀態篩選影響。
- [ ] 同時間多場以單一時間群組顯示，影廳自然排序。
- [ ] 倒數以實際 Date 差值更新，背景恢復後立即校正。
- [ ] waiting、playing、finished 邊界正確。
- [ ] 所有場次結束後不顯示負數、undefined 或 Invalid Date。

## 五、警報

- [ ] Desktop 啟動時鬧鐘預設開啟，沒有場次時不播放。
- [ ] 同時間多場只觸發一次警報。
- [ ] 最小化或被其他視窗覆蓋時，到點自動還原、置頂、聚焦及播放。
- [ ] Alarm Modal 的影廳、片名、語言與規格正確。
- [ ] 停止警報只停止目前群組，下一場仍會排程。
- [ ] 關閉鬧鐘後不補響已處理群組。
- [ ] 睡眠恢復與解除鎖定後以實際時間重新檢查。
- [ ] alarm.wav 錯誤時 Modal 仍正常顯示並提供非阻塞訊息。

## 六、DCP 中文片名

- [ ] 匯入實際 DCP 後 Schedule、Next Movie、Alarm Modal 與搜尋立即更新。
- [ ] 重複匯入新版完整取代舊 Map，不累加資料。
- [ ] 匯入失敗保留上一份有效 Map。
- [ ] 清除 DCP 後恢復英文片名且不重建日期、倒數或警報。
- [ ] 重啟程式後 DCP Map 自動恢復。

## 七、Windows Desktop

- [ ] Portable 可直接啟動，不建立永久登入啟動項目。
- [ ] Setup 安裝無 NSIS integrity error，可正常解除安裝。
- [ ] 開始功能表、桌面捷徑與工作列釘選可重新啟動程式。
- [ ] 正式圖示在 EXE、捷徑、視窗與系統匣正確顯示。
- [ ] 最小化後隱藏至系統匣，雙擊圖示可還原。
- [ ] 有 waiting／playing 場次時關閉會顯示「是／否」確認。
- [ ] Windows 登入啟動後常駐系統匣，不重複建立執行個體。
- [ ] 匯入場次後結束並重啟，資料恢復且過期場次不補響。
- [ ] 明確結束後沒有 Electron 程序殘留。

## 八、UI、相容性與無障礙

- [ ] Light／Dark Mode 的日期、文字、Badge、Focus 與 Modal 對比正常。
- [ ] 1440×900、1100×700 與窄螢幕沒有重要內容截斷。
- [ ] 搜尋、全部篩選、無結果與空狀態正確。
- [ ] 所有按鈕、選單與 Modal 可使用鍵盤操作。
- [ ] DevTools Console 沒有紅色錯誤或未處理 Promise。
- [ ] 斷網啟動時不請求 SheetJS、PDF.js 或其他執行依賴 CDN。
